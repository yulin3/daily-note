创建SSH Key ，进入git-bash界面

ssh-keygen -t rsa -C "注册邮箱"

在个人用户目录下生成私钥和公钥，将公钥加入github账户的SSH Key 使其允许推送

克隆仓库

git clone git@github.com:accountname/libraryname.git

默认情况下clone只能看到master分支，添加其他分支则

git checkout -b [localBranchName] origin/[originBranckName]


关联远程仓库，github先新建一个库 ，如果有.readme\ignore文件 先要同步一下本地

git pull

本地文件夹初始化

git init

关联远程仓库，origin 就是远程仓库名

git remote add origin git@github.com:accountname/libraryname.git

查看远程库信息

git remote -v

查看工作区状态

git status

将文件添加到暂存区

git add filename || git add -A

将文件提交到暂存区，如果没加参数-m 会进入到多行注释编写界面

git commit -m "comment"

推送文件到远程库，首次提交要加-u参数 将远程和本地的master关联起来

git push -u origin master

若没关联远程库分支则

git branch --set-upsteam [localBranchName] origin/[originBranchName]

查看文件记录

git diff file

查看日志记录

git log

记录每次命令，可以查看版本id

git reflog

回退版本，HEAD表示当前版本， ^表示上一个版本 || HEAD~1 ||  版本id

git reset --hard HEAD^ || HEAD~1 || commit id

撤销修改，--参数为必须，只能撤销未提交的版本

git checkout -- file

已经add或commit到暂存区则用

git reset HEAD file

删除文件

git rm file
 
误删除文件，远程库还存在，恢复用

git checkout -- file

查看分支 || 创建分支 || 创建并切换到分支 || 切换分支

git branch || git branch [name] || git checkout -b [name] || git checkout [name]

合并分支 || 默认合并是快速合并，不使用快速合并并创建一个新的提交

git merge [name] || git merge --no-ff-m [name]

删除分支 || -D 强行删除

git branch -d [name] || git branch -D [name]

缓存现场工作
git stash

查看缓存的文件列表

git stash list

恢复 || 恢复并删除 缓存的列表

git stash apply || pop [listid]

删除缓存列表

git stash drop [listid]

标签，有name则创建标签，没有则查看

git tag [name]

查看commit id

git log --pretty=online --abbrev-commit
 
对提交打标签

git tag [tag] [commit id]

加注释

git tag -a [tag] -m [comment]

查看标签信息

git show [tag]

删除标签

git tag -d [tag]

推送到远程库 || 推送所有未推送的

git push origin [tag] || git push origin --tags

删除远程库上的标签

git push origin :refs/tags/tagname

别名

git config --global alias.别名 原本命令

注释的规范 类型(影响范围 )：简单描述 空一行 详细描述 空一行 footer，body和footer非必需

<type>(<scope>):<subject>/<boby>/<footer>

注释工具 Commitizien 安装并执行 在执行commit的时候注释就换成cz的格式

npm install -g commitizen && commitizen init cz-conventional-changelog --save --save-exact

根目录下创建.gitignore文件，填入文件名，push时就会忽略这些文件
查看被过滤规则限制的文件则

git check-ignore -v [file]
